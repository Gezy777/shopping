## 一、压测背景与目标

### 1.1 压测背景

GoMall 是基于 Go 语言开发的微服务电商系统，包含以下核心服务：
- **frontend**: 前端网关服务（端口 8888）
- **user**: 用户服务（端口 8086）
- **product**: 商品服务（端口 8080）
- **cart**: 购物车服务（端口 8081）
- **order**: 订单服务（端口 8082）
- **checkout**: 结算服务（端口 8083）
- **payment**: 支付服务（端口 8084）
- **email**: 邮件服务（端口 8085）

### 1.2 压测目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 系统QPS | >50,000 | 峰值时刻目标 |
| 平均响应时间 | <100ms | 首页/商品列表 |
| P99延迟 | <200ms | 核心链路 |
| 订单成功率 | >99.5% | 不出现超卖等问题 |
| 错误率 | <0.1% | 无系统级错误 |

### 1.3 核心压测场景

1. **用户登录注册** - 验证用户认证流程
2. **商品浏览** - 首页+商品列表+商品详情
3. **购物车操作** - 添加/修改/删除购物车商品
4. **订单创建** - 提交订单流程
5. **支付流程** - 模拟支付回调

### 2.2 流量模型设计

根据电商业务特点，设计以下流量比例：

| 场景 | 占比 | QPS估算 | 说明 |
|------|------|---------|------|
| 首页/商品浏览 | 60% | 30,000 | 高频读操作 |
| 商品详情 | 20% | 10,000 | 中频读操作 |
| 购物车操作 | 10% | 5,000 | 写操作 |
| 下单流程 | 7% | 3,500 | 核心写操作 |
| 支付回调 | 3% | 1,500 | 回调操作 |

### 2.3 阶梯加压策略

```
阶段1: 预热阶段 (0-5分钟)
  - 并发用户: 100
  - 目标: 系统预热，观察各指标

阶段2: 增量压测 (5-20分钟)
  - 每2分钟增加20%用户
  - 100 → 120 → 144 → 173 → 207 → 248 → 298 → 358

阶段3: 峰值测试 (20-35分钟)
  - 维持最大并发 400 用户
  - 验证系统稳定性

阶段4: 突发测试 (35-40分钟)
  - 瞬间拉高到 600 用户
  - 测试系统突发能力

阶段5: 恢复测试 (40-45分钟)
  - 逐步降低到 100 用户
  - 验证系统恢复能力
```

## 三、压测脚本设计

### 3.1 wrk/Lua 压测脚本

#### 3.1.1 用户登录脚本
```lua
-- login.lua
wrk.headers["Content-Type"] = "application/json"

counter = 0
user_count = 10000

function request()
   counter = counter + 1
   user_id = counter % user_count + 1

   local body = string.format([[{
      "username": "user%d",
      "password": "password123"
   }]], user_id)

   return wrk.format("POST", "/api/user/login", wrk.headers, body)
end
```

#### 3.1.2 商品浏览脚本
```lua
-- product_browse.lua
wrk.headers["Content-Type"] = "application/json"

product_count = 1000
```

#### 3.1.5 全链路压测脚本
```lua
-- full_chain.lua
-- 模拟用户完整购物流程

wrk.headers["Content-Type"] = "application/json"

local token = nil
local user_id = math.random(1, 10000)
local product_id = math.random(1, 1000)
local order_id = nil

-- 阶段标记
local phase = "login"

function request()
   if phase == "login" then
      local body = string.format([[{
         "username": "user%d",
         "password": "password123"
      }]], user_id)
      return wrk.format("POST", "/api/user/login", wrk.headers, body)

   elseif phase == "browse" then
      path = string.format("/api/product/%d", product_id)
      return wrk.format("GET", path, wrk.headers)

   elseif phase == "cart" then
      local body = string.format([[{
         "product_id": %d,
         "quantity": %d
      }]], product_id, math.random(1, 3))
      wrk.headers["Authorization"] = "Bearer " .. (token or "test")
      return wrk.format("POST", "/api/cart/items", wrk.headers, body)

   elseif phase == "order" then
      local body = string.format([[{
         "items": [{"product_id": %d, "quantity": 1}],
         "address_id": 1,
         "payment_method": "alipay"
      }]], product_id)
      return wrk.format("POST", "/api/order/create", wrk.headers, body)

   elseif phase == "pay" then
      if order_id then
         local body = string.format([[{
            "order_id": "%s",
            "payment_id": "pay_%d"
         }]], order_id, math.random(1, 1000000))
         return wrk.format("POST", "/api/payment/callback", wrk.headers, body)
      end
   end

   -- 默认返回首页请求
   return wrk.format("GET", "/", wrk.headers)
end

function response(status, headers, body)
   if phase == "login" and status == 200 then
      -- 提取token
      token = "extracted-token-" .. user_id
      phase = "browse"
   elseif phase == "browse" then
      phase = "cart"
   elseif phase == "cart" then
      phase = "order"
   elseif phase == "order" and status == 200 then
      -- 提取order_id
      order_id = "order_" .. math.random(1, 100000)
      phase = "pay"
   end
end
```

## 四、执行步骤

### 4.1 前置检查

```bash
# 1. 检查所有服务状态
docker ps | grep gomall

# 2. 检查基础设施
curl -s http://localhost:8500/health
curl -s http://localhost:8888/ping

# 3. 检查数据库连接
mysql -h localhost -u gomall -p -e "SELECT 1"

# 4. 检查Redis
redis-cli ping
```

### 4.2 压测执行命令

```bash
# ===== 阶段1: 基础压测 =====

# 1.1 Ping接口基准测试
wrk -t8 -c200 -d60s --latency http://localhost:8888/ping

# 1.2 首页压测
wrk -t8 -c200 -d60s --latency http://localhost:8888/

# ===== 阶段2: 核心链路压测 =====

# 2.1 用户登录
wrk -t4 -c100 -d30s -s login.lua http://localhost:8888

# 2.2 商品浏览
wrk -t8 -c200 -d60s -s product_browse.lua http://localhost:8888

# 2.3 购物车操作
wrk -t4 -c100 -d30s -s cart_operation.lua http://localhost:8888

# 2.4 订单创建
wrk -t4 -c100 -d30s -s create_order.lua http://localhost:8888

# ===== 阶段3: 全链路压测 =====

# 3.1 完整购物流程
wrk -t8 -c200 -d300s -s full_chain.lua http://localhost:8888

# ===== 阶段4: 高并发测试 =====

# 4.1 极限QPS测试
wrk -t16 -c500 -d60s --latency http://localhost:8888/ping

# 4.2 混合场景压测
wrk -t8 -c300 -d120s --latency http://localhost:8888/
```

### 5.2 关键告警阈值

| 指标 | 警告值 | 严重值 | 处理措施 |
|------|--------|--------|----------|
| 错误率 | >0.5% | >1% | 停止压测排查 |
| P99延迟 | >300ms | >500ms | 检查瓶颈 |
| CPU使用率 | >80% | >90% | 扩容或优化 |
| 内存使用 | >85% | >95% | 检查内存泄漏 |
| QPS下降 | >20% | >30% | 停止压测分析 |
